    4. In `explainability.rs`, add new types and handler:
       - Define `ExplanationResponse` struct:
         ```rust
         #[derive(Debug, Clone, Serialize, Deserialize)]
         pub struct ExplanationResponse {
             pub action_id: String,
             pub decision_point: String,
             pub chosen_approach: String,
             pub alternatives_considered: Vec<AlternativeConsidered>,
             pub reasons: Vec<String>,
             pub source: String, // "causal_trace", "episodic", "negative_knowledge", "fallback"
         }

         #[derive(Debug, Clone, Serialize, Deserialize)]
         pub struct AlternativeConsidered {
             pub description: String,
             pub rejection_reason: Option<String>,
         }
         ```
       - Implement `pub(super) async fn handle_explain_action(&self, action_id: &str, step_index: Option<usize>) -> ExplanationResponse` on AgentEngine:
         1. Look up causal traces for the action_id (treat as goal_run_id). Query the WORM telemetry ledger for traces with matching goal_run_id. If `step_index` is provided, filter to traces at that step.
         2. If causal traces found: extract decision_point from `decision_type`, chosen_approach from `selected.reasoning`, alternatives from `rejected_options`, reasons from `causal_factors`. Return with source="causal_trace".
         3. If no causal traces: fall back to episodic retrieval. Query episodes for the goal_run_id topic. Extract relevant context. Return with source="episodic".
         4. If no episodic matches: query negative knowledge constraints related to the goal. Return with source="negative_knowledge".
         5. If nothing found: return a helpful fallback response: "This action was taken based on the goal plan step instructions. No detailed decision trace was recorded." Return with source="fallback".
         - IMPORTANT: Always return SOMETHING per research Pitfall 4. Never return empty.

       - Add `AgentExplanation { explanation_json: String }` variant to DaemonMessage enum.

    6. In `crates/amux-daemon/src/server.rs`:

         ```rust
         let explanation = agent.handle_explain_action(&action_id, step_index).await;

         ```
