    2. Create `crates/amux-daemon/src/plugin/oauth2.rs`:

       Define types:
       ```rust
       pub struct OAuthFlowConfig {
           pub client_id: String,
           pub client_secret: Option<String>,
           pub authorization_url: String,
           pub token_url: String,
           pub scopes: Vec<String>,
           pub pkce: bool,
       }

       pub struct OAuthFlowResult {
           pub access_token: String,
           pub refresh_token: Option<String>,
           pub expires_in: Option<u64>,
       }

       /// Holds the in-progress OAuth flow state between start and callback.
       pub struct OAuthFlowState {
           pub auth_url: String,
           pub redirect_uri: String,
           listener: tokio::net::TcpListener,
           csrf_state: String,
           pkce_verifier: Option<oauth2::PkceCodeVerifier>,
       }
       ```

       Implement functions:
       - `pub async fn start_oauth_flow(config: &OAuthFlowConfig) -> Result<OAuthFlowState>`:
         - Bind `TcpListener` on `127.0.0.1:0` per D-04.
         - Build redirect_uri as `http://127.0.0.1:{port}/callback` per D-05.
         - Create `BasicClient` with `ClientId`, optional `ClientSecret`, `AuthUrl`, `TokenUrl`, `RedirectUrl`.
         - Generate PKCE challenge if `config.pkce` is true: `PkceCodeChallenge::new_random_sha256()`.
         - Build authorization URL with `authorize_url(CsrfToken::new_random)`, add scopes, set PKCE challenge.
         - Return `OAuthFlowState` with auth_url, listener, csrf_state, pkce_verifier.

       - `pub async fn await_callback(state: &mut OAuthFlowState) -> Result<String>`:
         - Accept one TCP connection with 5-minute timeout per D-06.
         - Read HTTP request into 4096-byte buffer.
         - Parse the GET request line, extract path, use `url::Url::parse` to get code and state query params.
         - Validate state matches `state.csrf_state`. Reject on mismatch.
         - Send HTTP 200 response: "Authentication successful! You can close this tab."
         - Shutdown stream and return the authorization code.
         - Helper: `fn parse_callback_request(raw: &str) -> Result<(String, String)>` that extracts (code, state).

       - `pub async fn exchange_code(config: &OAuthFlowConfig, state: &OAuthFlowState, code: &str) -> Result<OAuthFlowResult>`:
         - Create `BasicClient` (same as start_oauth_flow).
         - Create reqwest client with `redirect(reqwest::redirect::Policy::none())` per Pitfall 4 (SSRF prevention).
         - Call `client.exchange_code(AuthorizationCode::new(code))` with optional `.set_pkce_verifier(verifier)`.
         - Use `.request_async(&http_client).await`.
         - Extract `access_token().secret()`, `refresh_token().map(|rt| rt.secret())`, `expires_in()`.

       - `pub async fn refresh_access_token(config: &OAuthFlowConfig, refresh_token_str: &str) -> Result<OAuthFlowResult>`:
         - Create `BasicClient` and no-redirect reqwest client.
         - Call `client.exchange_refresh_token(&RefreshToken::new(refresh_token_str)).request_async(&http_client).await`.
         - Return new `OAuthFlowResult`.

       - Add `#[cfg(test)] mod tests` with tests for `parse_callback_request` and auth URL construction.

       IMPORTANT: Never log token values. Use `tracing::info!` with only plugin name and success/failure per Pitfall 2.

    3. Add `start_oauth_flow_for_plugin()` method on `PluginManager`:
       ```rust
       pub async fn start_oauth_flow_for_plugin(&self, plugin_name: &str) -> Result<oauth2::OAuthFlowState>
       ```
       - Look up plugin, check it has `auth` section with `auth_type == "oauth2"`.
       - Get `client_id` and optionally `client_secret` from plugin settings (fetched via `self.persistence.get_settings()`).
       - Build `OAuthFlowConfig` from manifest auth section + settings.

    4. Add `complete_oauth_flow()` method on `PluginManager`:
       ```rust
       pub async fn complete_oauth_flow(&self, plugin_name: &str, state: &mut oauth2::OAuthFlowState) -> Result<()>
       ```
       - Call `oauth2::await_callback(state)` to get the authorization code.
       - Build `OAuthFlowConfig` again from plugin settings/manifest.
       - Call `oauth2::exchange_code(&config, state, &code)` to get tokens.
       - Load encryption key via `crypto::load_or_create_key()` using `self.plugins_dir.parent()`.
       - Encrypt access_token and store via `self.persistence.upsert_credential(plugin_name, "access_token", &encrypted, expires_at)`.
       - If refresh_token present, encrypt and store as `"refresh_token"` credential.
       - Compute `expires_at` as ISO8601 timestamp: `Utc::now() + Duration::seconds(expires_in)`.

    5. Extend `api_call()` in `mod.rs` per D-08/D-11:
       - After fetching settings (step e), check if plugin has OAuth auth:
         ```rust
         let auth_context = if plugin_has_oauth {
             self.get_oauth_context_with_refresh(plugin_name, &manifest_auth).await?
         } else {
             None
         };
         ```
       - Implement `get_oauth_context_with_refresh()`:
         - Load encryption key, get access_token credential from persistence.
         - If no credential, return `AuthExpired` error.
         - Decrypt access_token. Check expires_at against current time.
         - If within 80% of TTL (per D-08): acquire per-plugin refresh lock, check again (double-check after lock), refresh if still needed.
         - On refresh success per D-10: re-encrypt new tokens, upsert credentials.
         - On refresh failure per D-09: update credential type to mark expired, return `AuthExpired`.
         - Return `Some(auth_map)` with `"access_token"` and optionally `"refresh_token"` as strings.
       - Pass `auth_context` to `build_context()` as the third parameter.
       - Per D-12/AUTH-05: tokens only exist in local variables, never sent over IPC.

    6. Wire `PluginOAuthStart` handler in `server.rs`:
       - In the client message match block, add handler for `ClientMessage::PluginOAuthStart { name }`.
       - The handler should:
         a. Call `plugin_manager.start_oauth_flow_for_plugin(&name)` to get the flow state and auth URL.
         b. Send `DaemonMessage::PluginOAuthUrl { name, url: state.auth_url.clone() }` back to the requesting client.
         c. Spawn a `tokio::spawn` background task that:
            - Calls `plugin_manager.complete_oauth_flow(&name, &mut state)`.
            - On success: broadcasts `DaemonMessage::PluginOAuthComplete { name, success: true, error: None }` to all clients.
            - On failure: broadcasts `DaemonMessage::PluginOAuthComplete { name, success: false, error: Some(err.to_string()) }`.
         d. The spawned task needs access to the broadcast sender and plugin_manager Arc.
       - Follow the existing handler pattern (e.g., `PluginEnable`, `PluginTestConnection`).
       - Also handle the `PluginOAuthComplete` match arm in the daemon message exhaustive match (if needed for server-to-server forwarding -- likely a no-op match arm since it's only sent by daemon).
